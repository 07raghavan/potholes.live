# Contributing to potholes.live

Thank you for your interest in contributing! We welcome pull requests, issues, and suggestions to improve the project.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Make your changes** with clear, descriptive commit messages.
3. **Test your changes** locally before submitting.
4. **Submit a pull request** with a detailed description of your changes and the motivation behind them.

## Guidelines

- Follow the existing code style and structure (TypeScript, React, Tailwind CSS).
- Avoid introducing breaking changes unless necessary. If so, explain the impact in your PR.
- Do not include sensitive information, secrets, or credentials in your code or comments.
- For new features, update documentation and add usage instructions if needed.
- For bug fixes, describe the issue and how your fix resolves it.
- Keep the README and other docs up to date with your changes.

## Reporting Issues

- Use GitHub Issues to report bugs, request features, or ask questions.
- Provide clear steps to reproduce bugs and include relevant logs or screenshots.
- Suggest improvements with a clear rationale and possible implementation ideas.

## Code of Conduct

Be respectful, constructive, and collaborative. We aim to foster a welcoming and inclusive environment for all contributors.
